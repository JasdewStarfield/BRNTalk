# BRNTalk

**BRNTalk** 是一个为 Minecraft（NeoForge）设计的剧情对话模组，适合 RPG 服务器、任务地图与整合包使用。  
它专注于把“剧情阅读体验”做得更清晰、更沉浸：你可以把任务引导、世界观叙事、NPC 分支交流整合到统一的对话系统里。

---

## 你可以用它做什么？

- 构建 **NPC 剧情对话** 与 **多分支任务引导**
- 为玩家提供更统一的 **消息中心式 UI** 阅读体验
- 在关键节点做“暂停等待”，用于配合任务条件推进
- 通过可选提示模式（HUD / Toast / 关闭）控制提醒风格
- 与常见客户端生态（如 Cloth Config、JEI、FTB Library）更自然地协同

> 如果你想了解完整的脚本结构、命令、JSON 格式与开发 API，请查看仓库内 README。

---

## 功能亮点


- **IM-style UI 设计**：界面更接近即时通讯软件的信息流阅读体验，而不是 Galgame 的立绘+大对话框，也不是直接点世界实体弹出交互。
- **AI 协作开发**：架构设计与部分代码编写阶段使用了 AI 工具辅助，以提高迭代效率与一致性。
- **多线程会话管理**：同一玩家可拥有多条独立对话进度
- **沉浸式阅读 UI**：专用对话界面 + 新消息提示
- **分支与等待节点**：更适合任务链和阶段式剧情
- **可配置体验**：打字速度、滚动手感、UI 风格、提示位置等
- **服务端剧情友好**：面向多人服的剧情流程管理思路

## 快速开始指引（平台版）

1. 安装 BRNTalk 并进入游戏。
2. 确认客户端能打开 BRNTalk 的消息/对话界面。
3. 载入示例或你自己的对话内容资源。
4. 在游戏内触发一段对话，体验完整流程：
   - 自动推进文本
   - 分支选项
   - 等待节点后继续
5. 根据你的玩法风格，调整 UI 与提示配置。

> 想进一步自定义剧情、节点与联动逻辑：请直接参考仓库 README 的详细文档。
